# 第01天学生任务验收清单

## 基础运行验收

- [ ] 项目可以正常安装依赖并启动开发服务器。
- [ ] 浏览器可以打开项目页面，页面不白屏。
- [ ] 控制台没有影响继续开发的明显错误。

## 任务1：项目初始化与工程规范

验收重点：项目是否具备继续开发的工程基础。

- [ ] 项目结构清晰，能看出入口文件、样式目录和基础配置的位置。
- [ ] Vue 3 + JavaScript + Vite 项目能够正常运行。
- [ ] Element Plus、Pinia 等基础依赖已经接入并能正常使用。
- [ ] 全局样式和基础变量已经建立，页面有统一的基础视觉风格。
- [ ] README 或课堂说明中能指导同学启动项目。

## 任务2：后台主布局与基础路由

验收重点：后台应用的页面框架是否成型。

- [ ] 页面具备企业后台常见的侧边栏、顶部栏、面包屑和内容区域。
- [ ] 侧边栏菜单可以展示主要入口，并支持基础交互。
- [ ] 登录页、首页工作台、403、404 等基础页面可以正常访问。
- [ ] 路由跳转、浏览器刷新和错误路径访问不会导致白屏。
- [ ] 页面在常见屏幕宽度下没有明显布局错乱。

## 任务3：模拟接口与接口文档页

验收重点：前端是否能在没有真实后端的情况下完成接口联调闭环。

- [ ] 项目已经具备模拟数据和模拟接口能力。
- [ ] 健康检查接口和数据重置接口可以正常调用。
- [ ] 接口响应格式基本统一，便于后续业务页面复用。
- [ ] `/api-docs` 页面可以展示当前接口，并支持在线发送请求。
- [ ] 页面能够展示接口响应结果，我能说清楚请求从页面到 mock handler 的基本流程。

## 任务内选做验收

以下内容不作为最低完成要求，只作为课堂时间充裕时的加分项。

- [ ] 接口文档页支持 404 或 500 异常接口调试。
- [ ] 接口文档页包含请求参数或响应示例说明。
- [ ] 我能够画出 Layout 与 RouterView 的页面结构关系。
- [ ] 我能够说明后续客户管理模块需要哪些接口。
