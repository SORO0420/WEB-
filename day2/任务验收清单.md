## 任务1：认证模拟接口与用户状态管理

验收重点：登录是否真正通过接口完成，而不是页面直接跳转。

- [ ] 已创建 5 个固定演示账号，并包含角色、姓名、邮箱、手机号、头像和最近登录时间。
- [ ] 已创建角色、菜单和按钮权限编码数据。
- [ ] 已实现 `POST /api/auth/login`。
- [ ] 已实现 `POST /api/auth/logout`。
- [ ] 已实现 `GET /api/auth/profile`。
- [ ] 登录成功后页面可以进入工作台。
- [ ] 错误密码会显示错误提示。
- [ ] 用户 Store 能保存 Token、用户资料、菜单和权限。
- [ ] 登录页不直接导入模拟数据数组。

## 任务2：请求封装与认证路由守卫

验收重点：页面访问是否受登录状态和角色控制。

- [ ] Axios 实例设置了 `baseURL` 和超时时间。
- [ ] 请求拦截器会自动携带 `Authorization: Bearer <token>`。
- [ ] 401 响应会清理本地认证状态。
- [ ] Token、用户、菜单和权限能够持久化到 localStorage。
- [ ] 未登录访问 `/dashboard` 会跳转到 `/login?redirect=/dashboard`。
- [ ] 已登录访问 `/login` 会返回工作台。
- [ ] 页面刷新后能够通过 profile、routes 和 permissions 恢复状态。
- [ ] 无权限访问会进入 403 页面。
- [ ] 退出登录后不能通过浏览器返回继续访问后台。

## 任务3：动态菜单与按钮权限控制

验收重点：不同角色是否看到不同菜单和不同按钮。

- [ ] 侧边栏菜单来自 `GET /api/auth/routes`。
- [ ] `admin` 可以看到全部菜单。
- [ ] `sales`、`support`、`viewer` 至少三种角色菜单不同。
- [ ] 业务路由配置了角色限制，不能通过手输 URL 越权。
- [ ] 已实现 `hasPermission` 或等价权限判断函数。
- [ ] 工作台至少有 3 个按钮权限示例。
- [ ] `viewer` 看不到操作按钮。
- [ ] `support` 可以看到工单相关入口和按钮。
